Propsâ€‹

dataâ€‹

PropTypeOptional
dataarray of objectsfalse

xKeyâ€‹

This string is used as the accessor for your data in the X domain. This should match the property name from your data that you want to use for your X values. For example, if we want to show the quantity of fruit over time using the above dataset, 'date' will be our xKey value.

PropTypeOptionalRequirements
xKeystringfalseA key corresponding to the properties in array of data objects used on x axis.

yKeyâ€‹

This string is used as the accessor for the Y values of your data. This should match the property name from your data that you want to use for your Y values. For example, if we want to show the quantity of fruit using the above dataset, 'value' will be our yKey value.

PropTypeOptionalRequirements
yKeystringfalseA key corresponding to the properties in array of data objects used on y axis.

xDataType (optional)â€‹

By default, D3Reacts will attempt to infer whether the data accessed by xKey is a valid date. (If it is, it will use D3's scaleTime method to create the X axis rather than scaleLinear. This helps create more readable X axis ticks.) By passing in a value to xDataType, you can manually set how you want D3Reacts to interperet your data.

PropTypeOptionalDefault
xDataType`"number""date"undefined`

groupBy (optional)â€‹

To create a multi-area chart, pass in a string to groupBy. This string should match the property name from your data that represents the distinct groups you want to show. For example, in the above dataset our groups are stored in the property name 'fruit'.

PropTypeOptionalDefaultRequirements
groupBystringtrueundefinedA key corresponding to the properties in array of data objects used to denote groups.

legend (optional)â€‹

Each chart has the option to include a legend which shows what category from your data each color represents. The legend and chart are both responsive, so they will resize to make space for each other. The categories reflect the list of unique values stored at the property in your dataset accessed by groupBy.

PropTypeOptionalDefault
legendboolean or "top", "bottom", "left", "right" , "top-left", "top-right", "bottom-left", "bottom-right", "left-bottom", "right-bottom", "left-top", "right-top"truefalse

If true, the legend will default to position right.

legendLabel (optional)â€‹

This is the title at the top of the legend. If the value is an empty string (''), the legend label will not render.

PropTypeOptionalDefault
legendLabelstringtrue''

You must specify a legend property if you wish to use legendLabel

colorSchemeâ€‹

Sets the color scheme of the chart body. Accepts an array of strings, each representing a color (e.g. 'red', 'rgb(63,63,63)', '#9dc8e2', etc.). The order of colors should match the order of unique groups accessed by groupBy in your dataset.

PropTypeOptionalDefault
colorSchemearray of stringstrue8 colors between "#9dc8e2" and "#07316b"

If the number of unique groups exceeds the number of colors in the array, the colors will loop, repeating in order until all groups are colored.
